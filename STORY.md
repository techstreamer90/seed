## The Concepts (for normal humans)

### 1. "Reality is Part of the Model"
Instead of documentation that gets outdated, the model literally POINTS to the real files with checksums. Change the file? The model screams "DRIFT DETECTED!"

### 2. "Intent vs Reality"
- Thing without a source reference = "we want this to exist"
- Thing with a source reference = "this exists, here's proof"
- The gap between them = the actual work to do

### 3. "Self-Containing Models"
Each project contains its own model. The model lives inside what it describes. It's turtles all the way down.

### 4. "Model as Workspace"
